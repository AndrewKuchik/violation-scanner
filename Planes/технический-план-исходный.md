# Violation Scanner — сканер нарушений cookie/GDPR с оценкой штрафа

## Контекст

Идея: сайт, куда пользователь вставляет любой URL, и получает отчёт о нарушениях законов ЕС/Латвии (GDPR, ePrivacy) — **что** нарушено, **где** именно на сайте, и **оценочный диапазон** финансовых последствий, если не исправить. Это прототип для проверки идеи (не коммерческий продукт), но детектор нарушений и логика оценки риска должны быть содержательными — это и есть вся ценность продукта.

Перед проектированием был проведён углублённый ресёрч рынка (18+ подтверждённых источников: Cookiebot, CookieYes, Osano, FineMyData, EDPB guidelines, реальные штрафы DVI Latvia, WebAIM). Ключевые выводы, определившие архитектуру:

- **Рыночная ниша подтверждена**: ни один существующий инструмент не сочетает (а) реальное сканирование URL в браузере, (б) визуальную локализацию "где" на странице, и (в) оценку финансового риска. Cookiebot/CookieYes/Osano сканируют, но не оценивают штраф. FineMyData оценивает штраф, но через анкету, а не скан URL — не может показать "где".
- **Юридически критично**: EDPB (регулятор) прямым текстом пишет, что расчёт штрафа — не механическое упражнение, итог зависит от 11 факторов (ст. 83(2)), которые сканер физически не может считать с URL (умысел, история нарушений, сотрудничество с властями и т.д.). Реальный пример — латвийская DVI по делу SIA "TET": стартовая сумма €3.2M снижена до €1.2M просто за сотрудничество после обнаружения. **Вывод: система должна выдавать диапазон + факторы, никогда не точную цифру.**
- **Юридический анти-паттерн, которого нужно избежать**: accessiBe получил от FTC штраф $1M и классовый иск за заявление "наш виджет = гарантированное соответствие закону". **Продукт никогда не должен утверждать "вы теперь соответствуете закону"** — только "эти конкретные проблемы устранены".
- Пользователь подтвердил: v1 = только детекция cookie/consent (GDPR+ePrivacy), доступность (EAA) — в v2. AI-слой (объяснение находок человеческим языком) — включить сразу.

## Технологический стек

- **Next.js (App Router, TypeScript)** — единое приложение, без отдельного backend
- **Playwright (chromium)** — headless-браузер для рантайм-сканирования (важно: должен ловить куки/трекеры, которые грузятся ДО согласия — это и есть основное нарушение)
- **Anthropic TypeScript SDK**, модель **`claude-opus-4-8`** — генерация понятных объяснений находок со ссылкой на статью закона. Structured output через `output_config.format` (JSON Schema), не через forced tool_use — так проще валидировать и не даёт модели "изобретать" цитаты закона (они передаются как статичные данные из правил, а не генерируются моделью)
- **Tailwind** — UI отчёта
- **Без очереди/БД в v1.** Next.js работает как обычный долгоживущий Node-процесс (`next build && next start`), НЕ на serverless-функциях — у Playwright там проблемы с размером Chromium-бинарника и холодным стартом. Обычный процесс не имеет лимита времени запроса, скан на 10-30 сек просто отработает. Отчёт возвращается как JSON и рендерится на клиенте — персистентность не нужна для прототипа с ручными единичными сканами.

## Архитектура

```
violation-scanner/
  src/
    app/
      page.tsx                    # форма ввода URL + уточняющие вопросы
      api/scan/route.ts           # POST — оркестрирует весь пайплайн, Node.js runtime
      report/page.tsx             # рендер отчёта (клиентский fetch)
    lib/scanner/
      browser.ts                  # запуск/остановка chromium
      capture.ts                  # навигация + подписка на события ДО goto() — критично для перехвата "до согласия"
      evidence/
        cookies.ts                # снимок cookies через context.cookies()
        storage.ts                # localStorage/sessionStorage
        network.ts                # лог запросов + сверка с базой известных трекеров
        trackers.ts                # статический список доменов (GA, GTM, Meta Pixel, ...)
        consentBanner.ts          # эвристика детекции банера (селекторы известных CMP + текстовые паттерны), с поддержкой iframe/shadow-DOM
        links.ts                  # наличие/доступность Privacy/Cookie Policy
        tls.ts                    # https + mixed content
      types.ts                    # ScanEvidence, Finding, Report, Rule
      rules/
        index.ts                  # реестр правил + runRules(evidence): Finding[]
        consent/*.ts               # 8 правил v1 (см. ниже)
      scoring/
        severity.ts               # правило+evidence → critical|high|medium|low
        fineRange.ts               # правило+severity+companySizeTier+repeatOffender → диапазон €
      ai/
        explain.ts                 # один батч-вызов Claude на весь скан
        prompts.ts                  # системный промпт + JSON-схема
      report/buildReport.ts        # финальная сборка Report
    components/report/
      ReportSummary.tsx, FindingCard.tsx, SeverityBadge.tsx,
      EvidencePanel.tsx, FineRangeBox.tsx, DisclaimerBanner.tsx
    components/ScanForm.tsx, QualifyingQuestions.tsx
```

**Поток**: `POST {url, companySizeTier?, repeatOffender?}` → запуск chromium → `capture.ts` подписывается на `page.on('request')`/`page.on('response')`/снимок cookies **до** `page.goto()` → сбор evidence → `rules/index.ts` прогоняет 8 правил → каждая находка получает severity + диапазон штрафа → один батч-запрос к Claude добавляет понятные объяснения (на основе статичных `legalRefs` из правила — модель не придумывает цитаты) → сборка `Report` → JSON-ответ.

## Правила v1 (все проверяемы по публичному URL без логина)

| Правило | Источник evidence | Базовая severity |
|---|---|---|
| Куки установлены до согласия | `cookies.ts`, снимок до любого взаимодействия | Critical |
| Известный трекер грузится до согласия (GA/GTM/Meta Pixel) | `network.ts` + `trackers.ts` | Critical |
| Банер согласия отсутствует полностью | `consentBanner.ts` | Critical |
| Банер: асимметрия трения (accept в 1 клик, reject — много) | `consentBanner.ts` | High |
| Банер: нет видимой кнопки отказа | `consentBanner.ts` | High |
| Банер: чекбоксы согласия предустановлены | `consentBanner.ts` | High |
| Отсутствует/недоступна ссылка на Privacy/Cookie Policy | `links.ts` | Medium |
| Нет HTTPS / mixed content | `tls.ts` | Medium-High |

Правила — обычные TS-функции с метаданными (`Rule` interface: `id`, `legalRefs`, `severityBase`, `evaluate()`), не JSON DSL — при 8-15 правилах в v1 DSL избыточен. Регистр в массиве позволяет добавлять `rules/accessibility/*.ts` в v2 без переписывания движка.

## Оценка финансового риска — самая юридически чувствительная часть

**Никогда не выдаётся единая точная сумма.** Всегда диапазон + список факторов, которые на него повлияли.

- Опорные точки (из ресёрча): реальные латвийские штрафы DVI — от €7 000 (2019, малый ритейлер, отягчено несотрудничеством) до €1 200 000 (2022, SIA "TET", крупная телеком-компания; изначально DVI запросила €3 200 000, снизила на 50% за сотрудничество). Статутные потолки GDPR ст. 83: нижний уровень до €10M/2% оборота, верхний — до €20M/4% (зависит от типа нарушения).
- Продукт задаёт 1-2 лёгких уточняющих вопроса, которые нельзя вывести из скана: примерный размер компании (соло/малый бизнес / SME / крупная компания) и разовое/повторное нарушение — с диапазоном на каждый уровень. По умолчанию (если пропущено) — консервативное предположение "малый бизнес, впервые".
- Обязательный, хорошо заметный disclaimer (по образцу FineMyData): "Это образовательная оценка, не юридическая консультация. Реальный штраф определяет DVI или другой DPA индивидуально."
- **Нигде не должно быть формулировки "исправление = гарантированное соответствие закону"** — только "проблема устранена" / "риск снижен".

## AI-слой объяснений

- Один батч-запрос Claude на весь скан (не по одному вызову на находку — дешевле, быстрее, позволяет кэшировать общий системный промпт).
- Модель: `claude-opus-4-8`.
- `output_config: {format: {type: "json_schema", schema: ...}}` — структурированный JSON-вывод, который валидируется и мапится обратно на `Finding[]`.
- Модели передаются готовые `legalRefs` (статично прописаны в правиле) как контекст — задача модели только перефразировать понятным языком и дать шаги по исправлению, **не изобретать номера статей закона**.
- Системный промпт явно запрещает формулировки о гарантированном соответствии закону.

## UI отчёта

- `DisclaimerBanner` — закреплён вверху, не в подвале.
- `ReportSummary` — общее число находок по severity, общий диапазон риска (например "€8 000 – €45 000"), допущение о размере компании со ссылкой на изменение.
- `FindingCard`, сгруппированные по severity (critical сверху), свёрнуты по умолчанию: severity badge, заголовок, одна строка понятного описания. Разворачивается: полное объяснение, `EvidencePanel` (скриншот подсвеченного элемента для DOM-находок Playwright `locator.screenshot()`, либо структурированные данные для cookie/network-находок), цитата закона, `FineRangeBox`, шаги по исправлению.
- Локализация "где" — единственное, что не делает толком ни один конкурент: для DOM-находок (банер) — `locator.evaluate()` подсветка рамкой + `locator.screenshot()`, обрезанный по bounding box. Для cookie/скриптов — структурированные данные (имя, домен, момент появления).

## Порядок разработки (walking skeleton → полный пайплайн)

1. **Скелет**: Next.js + Playwright, один API route, возвращает `{title, cookies}` — подтвердить, что Playwright работает внутри обычного Node-процесса.
2. **Сбор evidence**: cookies/storage/network/trackers/consentBanner (эвристики под известные CMP: OneTrust, Cookiebot, Osano, generic-селекторы)/links/tls. Ручное тестирование на 3-5 реальных сайтах.
3. **Движок правил**: 8 правил v1, вывод `Finding[]` с severity, пока без диапазона штрафа.
4. **Калькулятор диапазона штрафа** + уточняющие вопросы UI, привязка к опорным точкам DVI.
5. **AI-слой объяснений**: один батч-вызов Claude, structured output, привязка к статичным `legalRefs`.
6. **Скриншоты/подсветка + финальный UI отчёта**.

## Риски

- **Бот-детекция**: некоторые сайты (Cloudflare и т.п.) блокируют headless Chromium — отчёт должен явно писать "скан может быть неполным", а не молча выдавать ложный "нарушений не найдено".
- **Банеры согласия в iframe/shadow DOM**: `consentBanner.ts` должен использовать `page.frameLocator()` и селекторы, пробивающие shadow-root, с честным "детекция банера неубедительна" вместо ложной уверенности.
- **Точность юридических цитат**: `legalRefs` — статичные, вручную прописанные в правилах данные, никогда не генерируются AI.
- **Тайминг "до согласия"**: слушатели должны быть подписаны до `page.goto()`; v1 не кликает "отклонить" автоматически (это отдельная более хрупкая фича) — фиксируется только состояние до любого взаимодействия, что уже покрывает самую частую категорию нарушений.

## Проверка (verification)

Ручное тестирование на реальных сайтах: 1) сайт с очевидным нарушением (GA грузится до согласия) — должен найтись как Critical; 2) сайт с более-менее compliant CMP — минимум ложных срабатываний; 3) прогнать через AI-слой и проверить, что объяснения не содержат придуманных статей закона и нет формулировок про "гарантированное соответствие"; 4) визуально проверить, что скриншот/подсветка элемента реально указывает на правильное место на странице.
