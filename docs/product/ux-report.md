# UX-принципы отчёта

Из анализа лучших практик рынка (WebAIM, аудит-отчёты, Osano).

1. **Сначала сводка, потом детали.** Нетехническому пользователю нужен «взгляд с высоты» перед списком.
2. **4-уровневая severity** (Critical / High / Medium / Low). Уровень = воздействие × распространённость, не сложность исправления.
3. **Локализация «где» — визуальная, не только текст.** Для DOM-находок скриншот с подсветкой рамкой. Это главный дифференциатор — конкуренты так не делают.
4. **Простой язык сначала, техника — по клику.** Каждая карточка: одна понятная строка видна сразу, детали при разворачивании.
5. **Конкретные шаги по исправлению**, не только констатация проблемы.
6. **Дисклеймер закреплён вверху** (не в подвале) — см. `legal-guardrails.md`.

## Экран сканирования
