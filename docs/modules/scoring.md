# Модуль: Severity + оценка штрафа (Ф.4)

⚠️ Читай вместе с `product/legal-guardrails.md` — здесь самые жёсткие юр-ограничители.

## Файлы
- `lib/scoring/severity.ts` — `правило + evidence → critical|high|medium|low`.
  Уровень = воздействие на пользователя × распространённость по сайту (НЕ сложность исправления).
- `lib/scoring/fineRange.ts` — `правило + severity + companySizeTier + repeatOffender → диапазон €`.

## Механика диапазона
- **Никогда не точная сумма** — только диапазон + факторы (см. guardrails).
- 1–2 уточняющих вопроса, которые нельзя вывести из скана:
  - размер компании: соло / малый бизнес / SME / крупная;
  - разовое / повторное нарушение.
- Пропущено → консервативно «малый бизнес, впервые».
- Диапазон привязан к откалиброванным «уровням экспозиции» на базе опорных точек DVI (см. guardrails §5), а не к точной формуле.

## Шкала severity в отчёте
4 уровня: Critical / High / Medium / Low. Critical — сверху.
