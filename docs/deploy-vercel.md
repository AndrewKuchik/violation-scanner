# Публикация в интернете (бесплатно, Vercel)

Цель: живая страничка по ссылке (например `violation-scanner.vercel.app`), которую
можно пересылать. Обновляется сама при `git push` в GitHub. Локальная разработка
при этом НИКУДА не девается — облако это просто вторая, публичная копия.

## Почему Vercel, а не GitHub Pages
Наш сканер на сервере **реально запускает браузер Chromium** (открывает чужой
сайт, ловит куки/трекеры). GitHub Pages умеет только статичные файлы и такое НЕ
запускает. Vercel — умеет (Node.js + serverless). Код при этом всё равно живёт на
GitHub; Vercel просто читает его оттуда и запускает.

## Как устроено в коде (уже готово)
- `lib/scanner/browser.ts` — два режима: локально обычный `playwright`; в облаке
  (`process.env.VERCEL`) — `playwright-core` + `@sparticuz/chromium` (лёгкая сборка
  chromium под serverless).
- `app/api/scan/route.ts` — `maxDuration = 60` (лимит функции Vercel Hobby);
  `SCAN_TIMEOUT_MS` на Vercel = 52 с (отдаём честную ошибку раньше, чем платформа
  убьёт функцию на 60-й секунде).
- `next.config.ts` — облачные пакеты в `serverExternalPackages` (не бандлить).

## Разовая настройка (единственные ручные шаги — нужен твой вход)
1. Зайти на **vercel.com** → **Sign Up** → **Continue with GitHub** (вход через твой
   GitHub-аккаунт). Это как `gh auth login` — только ты можешь это сделать.
2. **Add New… → Project** → в списке репозиториев выбрать **`violation-scanner`** →
   **Import**. (Если репозитория не видно — «Adjust GitHub App Permissions» и дать
   доступ к нему.)
3. Vercel сам определит Next.js. Ничего в Build settings менять не надо.
4. Раскрыть **Environment Variables** и добавить (рекомендуется):
   - `PLAYWRIGHT_SKIP_BROWSER_DOWNLOAD` = `1` — не тратить время сборки на скачивание
     ненужного в облаке браузера (там свой, sparticuz).
   - AI по умолчанию ВЫКЛ — ключи добавлять НЕ нужно. Если позже захочешь включить:
     `AI_EXPLANATIONS_ENABLED=true` и `ANTHROPIC_API_KEY=<ключ>`.
5. **Deploy**. Через ~2–3 минуты появится адрес вида `violation-scanner.vercel.app`.

## Проверка после первого деплоя
Открыть адрес → вставить URL сайта → «Проверить». Первый скан в облаке — это момент
истины для облачного браузера. Если сканирование падает с ошибкой про browser/
chromium — это ожидаемая точка доводки (трейсинг файлов sparticuz на Vercel иногда
требует правки). Пришли текст ошибки — доведём.

## Дальше
Каждый `git push origin master` → Vercel сам пересобирает и обновляет живой сайт.
Отдельно ничего деплоить не надо.

## Запасной вариант
Если облачный браузер на Vercel окажется капризным — переезжаем на **Render** (Docker
с официальным образом Playwright: браузер работает «как есть», без sparticuz).
Минусы Render free: 512 МБ памяти и «засыпание» после 15 мин простоя (первый заход
~минуту). Держим в уме, если понадобится.
